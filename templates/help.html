Help Page




Summary:

Non-Workflow Tasks page is designed to run various tasks for group deposition entries. It can run a task simultaneously on multiple entries of same group deposition as a batch. All entries of the group are selected by default. If you only want to update certain entries to be updated, click "Unselect All" button and select cetain entries by clicking the checkbox in the front of deposition ID.

The purpose of each task button are summarized in the following text. For "Merge Sequence Annotation", "Status Tasks", "Merge Other Values", and "Merge Initial Deposition Date/Revision History" tasks, more detail information are provided in each section.

"Merge Sequence Annotation" button: Merge sequence annotation information from a processed template entry.

"Status Tasks" button: Update status information.

"Merge Other Values" button: Merge other annotation information from a processed template entry.

"Cif Check" button: Run Cif Checking.

"Generate PDB Files" button: Generate PDB files.

"Load Database" button: Load entries into database.

"Copy Processed Files to Deposition" button: Copy processed files to deposition server.

"Merge Initial Deposition Date/Revision History" button: Copy back the initial deposition date and/or revision history categories (for released entries only) lost during depositor re-upload files. This task will be removed after the fixing is being deployed in production.

Merge Sequence Annotation:

This task merges sequence annotation information from a processed entry file with same author sequence(s) into selected entries of the group deposition.

"Fix ALA/GLY mismatch" checkbox: If checked, the ALA/GLY mismatch(es) between author sequence and cordinates will be automatically fixed during merge process.

Status Tasks:

This task updates status et al. information in "pdbx_database_status" category for selected entries of the group deposition.

"Processing Status Code": Update "_pdbx_database_status.status_code" value.

"Approval type": Update "_pdbx_database_status.author_approval_type" value.

"Author Release Status Code": Update "_pdbx_database_status.author_release_status_code" value.

"Author hold date": Update "_pdbx_database_status.date_hold_coordinates" value.

"Reassign annotator": Update "_pdbx_database_status.pdbx_annotator" value.

"Process site": Update "_pdbx_database_status.process_site" value.

Merge Other Values:

This task merges selected information from a processed entry file into selected entries of the group deposition. If only "Ligand of interesting" information needs to be updated, no template model file is needed.

"Header" checkbox: If checked, copy "_struct_keywords.pdbx_keywords" value from the template model file.

"Title" checkbox: If checked, copy "_struct.title" value from the template model file.

"Keywords" checkbox: If checked, copy "_struct_keywords.text" value from the template model file.

"Authors" checkbox: If checked, copy "audit_author" and "pdbx_contact_author" categories from the template model file.

"Group information" checkbox: If checked, copy "pdbx_deposit_group" category from the template model file.

"Citation" checkbox: If checked, copy "citation", "citation_author" and/or "citation_editor" categories from the template model file.

"Crystallization conditions" checkbox: If checked, copy "exptl_crystal_grow", and "exptl_crystal_grow_comp" categories from the template model file.

"Crystallization conditions" checkbox: If checked, copy "exptl_crystal_grow", and "exptl_crystal_grow_comp" categories from the template model file.

"Ligand of interesting" checkbox: Please run "Load Database" task first if the entries have been updated but not loaded into database yet before clicking "Ligand of interesting" checkbox. If checked, all ligand(s) associated with the entry (if exist) will show up next to the entry checkbox. You can use "Select All" button to select all ligands for all entries and manually select individual ligand for each entry. The program will update "pdbx_entry_details" category and add the selected ligand(s) into "pdbx_entity_instance_feature" category.

Merge Initial Deposition Date/Revision History:

Temporary solution to address some issues caused by depositor re-upload files after entries have been deposited or released.

"Initial Deposition Date" checkbox: If checked, copy "_pdbx_database_status.recvd_initial_deposition_date", "_pdbx_database_status.date_coordinates", "_pdbx_database_status.date_struct_fact", and "_pdbx_database_status.date_submitted" values from the first version of archival model file.

"Revision History" checkbox: If checked, copy revision history (pdbx_audit_revision_*) categories from last version of release milestone model file.
